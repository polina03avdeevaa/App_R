# Анализатор текста 

Shiny-приложение для лингвистического анализа русскоязычных текстов: облако слов, частотный анализ, биграммы, разбор по частям речи.

## Зачем это нужно

Юзер вставляет любой русскоязычный текст — главу книги, статью, эссе, серию постов — и сразу получает разноплановый разбор:

- **Базовая статистика** — количество слов, уникальных слов, предложений, лексическое разнообразие (TTR), средняя длина слова
- **Топ слов по частоте (TF)** — график и таблица
- **Облако слов** — визуализация частот с цветовой шкалой по интенсивности
- **Биграммы** — самые частые словосочетания
- **Анализ частей речи** — топ существительных, глаголов и прилагательных по отдельности

Все вычисления — на **леммах** (нормальных формах), а не на сырых словоформах. Это критично для русского: без лемматизации «дочка», «дочку», «дочери» считались бы тремя разными словами, а частота главных персонажей текста была бы размазана по падежам.

## Кому полезно

- Студентам и исследователям, работающим с текстами на русском
- Авторам блогов и эссе, которые хотят посмотреть на свой стиль со стороны
- Преподавателям филологии и языков для иллюстрации лингвистических концепций
- Всем, кто хочет провести быстрый предварительный разбор текста перед более глубоким анализом

## Архитектура

```
[Пользователь вставляет текст]
            │
            ▼
   [Кнопка "Проанализировать"]
            │
            ▼
  observeEvent → reactiveVal
            │
            ▼
     annotate_text()
   (udpipe лемматизирует
    ОДИН раз на клик)
            │
            ▼
        ann_data
       (reactive)
            │
   ┌────────┼────────┬──────────┬──────────┐
   ▼        ▼        ▼          ▼          ▼
word_freq get_ngrams get_pos analyze_tf text_stats
   │        │        │          │          │
   ▼        ▼        ▼          ▼          ▼
Облако  Биграммы  Части речи   TF       Статистика
        (plotly)   (ggplot)   (ggplot)    (HTML)
```

**Ключевое архитектурное решение:** тяжёлая лемматизация запускается один раз по нажатию кнопки, результат кэшируется в реактивном выражении и переиспользуется во всех вкладках. Движение слайдеров перерисовывает графики **без повторного прогона** udpipe.

## Основа: контекстная лемматизация

Без udpipe это приложение не имело бы смысла. На русском один токен встречается в десятках падежных и числовых форм. Если их не свести к нормальной форме:

- Частотные графики размазаны и нечитаемы
- Облако слов забивается вариантами одного слова
- Биграммы дублируются («капитанская дочка» / «капитанскую дочку»)
- Анализ частей речи невозможен в принципе

`udpipe` использует **контекстную лемматизацию** через нейросетевую модель `russian-syntagrus`. Это значит, что омонимы разрешаются по синтаксическому контексту: «стекла» как существительное (родительный падеж от «стекло») и «стекла» как глагол (форма от «стечь») получают разные леммы в зависимости от того, как они встроены в предложение.

## Особенности реализации

**1. Биграммы с двойным представлением.** Для подсчёта используется лемма-сигнатура («капитанский дочка»), для отображения на графике — самая частая исходная форма из встретившихся («капитанская дочка»). Это даёт и точный подсчёт, и читаемый результат.

**2. Биграммы внутри предложений.** Использован паттерн `group_by(sentence_id) |> mutate(lead(...))`, чтобы пары слов не пересекали границы предложений. Никаких ложных биграмм вроде «предложения начало» из «...конец предложения. Начало следующего...».

**3. Реактивное кэширование.** Аннотация текста (самый дорогой шаг) обёрнута в `reactive()`, что делает её ленивой и кэшируемой. Слайдеры в интерфейсе пересчитывают только лёгкие операции, тяжёлый udpipe не дёргается лишний раз.

**4. Кастомная тема.** `bslib::bs_theme()` с фирменной палитрой (кремовый фон `#D6D1BC`, оливковый акцент `#AFA449`, бордо `#770205` для заголовков) + Inter из Google Fonts. Палитра согласована между `bslib`-темой, `ggplot2`-графиками, `wordcloud2` и таблицами `DT`.

## Установка и запуск локально

```r
# 1. Установить зависимости
install.packages(c(
  "shiny", "bslib", "udpipe", "dplyr", "tidyr", "stringr",
  "tidytext", "stopwords", "ggplot2", "plotly", "wordcloud2",
  "DT", "showtext", "scales", "forcats"
))

# 2. Клонировать репозиторий
# git clone https://github.com/<user>/<repo>.git

# 3. Запустить приложение
shiny::runApp()
```

При первом запуске модель udpipe (~30 МБ) автоматически скачивается в рабочую папку, если её ещё нет.
